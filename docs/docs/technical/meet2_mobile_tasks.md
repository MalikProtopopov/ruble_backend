# Доработки мобильного приложения по итогам встречи 2

Источник: `docs/mocks/2_meet.md`. Бэкенд-часть: `meet2_backend_tasks.md`.

Документ описывает только мобильное приложение (Flutter). Для каждой задачи указаны новые/изменённые экраны и контракты API, которые мобилка должна потреблять.

---

## 1. Анонимная авторизация

### 1.1. Auto-register при первом запуске
- При первом запуске генерировать локальный `device_id` (UUID v4), сохранять в secure storage.
- При старте приложения, если нет access/refresh токена в storage → дёрнуть `POST /auth/device-register` с `device_id`, `push_token` (если уже выдан Firebase'ом), `push_platform`, `timezone`.
- Сохранить полученные access+refresh токены.
- Все последующие запросы идут с этим токеном — точно так же, как для зарегистрированного юзера.

### 1.2. Refresh без интервенции
- Refresh-токен у гостя живёт ~6 месяцев. На фронте логика обычная: 401 → дёрнуть `/auth/refresh` → повторить запрос.
- Если refresh умер → снова вызвать `device-register` с тем же `device_id`. Бэкенд вернёт того же юзера.

### 1.3. Экран «привязать почту» (опционально)
- Новый раздел в профиле «Сохранить аккаунт». Объясняет, что email нужен, чтобы не потерять историю при смене устройства.
- Поток: ввод email → `POST /auth/send-otp` → ввод кода → `POST /auth/link-email/verify-otp` (с текущим access токеном гостя).
- После успеха обновить локального юзера: `is_anonymous=false`, `email=...`. Показать тост «Аккаунт сохранён».
- Если бэкенд вернул, что email уже принадлежит другому юзеру → показать модалку «У вас уже есть аккаунт с этим email. Объединить?». При подтверждении → дёрнуть тот же endpoint с флагом `merge=true`. После успеха перелогинить пользователя в целевой аккаунт (новые токены придут в ответе).

### 1.4. Удалить блокеры в существующих экранах
- Убрать обязательность авторизации перед донатом. Сейчас (на встрече упоминалось) экран доната просит почту → надо снять это требование. Юзер уже залогинен через device-register, можно сразу платить.

## 2. Главный экран: список сборов

### 2.1. Новые поля в карточке сбора
Бэкенд начнёт возвращать в `GET /campaigns` (для авторизованных):
- `donated_today: bool`
- `last_donation: { id, amount_kopecks, created_at } | null`
- `next_available_at: ISO datetime | null`
- `has_any_donation: bool`

### 2.2. Визуал кнопки «Помочь»
- Если `donated_today == true` → кнопка в состоянии «Уже помог сегодня» (галочка, неактивный/disabled стиль).
- Если `next_available_at > now` → кнопка disabled с таймером «Можно через Xч Yм» (вычисляется на клиенте, обновляется раз в минуту).
- Иначе обычная активная «Поддержать».

### 2.3. Сортировка
Параметр `sort` в `GET /campaigns` (`default` / `helped_today` / `helped_ever`). Добавить переключатель/табы в UI. По умолчанию — `helped_today`.

### 2.4. Сброс «галочек» в полночь
Никакой клиентской логики не требуется — бэкенд считает `donated_today` по таймзоне юзера. Главное при pull-to-refresh снова дёрнуть список.

## 3. Фича «3 сбора на сегодня»

### 3.1. Новый блок на главной
- Сверху списка кампаний — виджет «Сегодня помогаем» с горизонтальным скроллом из 3 карточек.
- Источник: `GET /campaigns/today`.
- Каждая карточка: превью (`thumbnail_url`), название, прогресс, кнопка «Помочь» с теми же состояниями (см. 2.2).
- Чекбокс/галочка «помог сегодня» сбрасывается автоматически (бэкенд считает).

---

## 4. Экран сбора и cooldown

### 4.1. Обработка 429
При `POST /donations` бэкенд может вернуть `429` с телом `{ error: "donation_cooldown", retry_after, next_available_at }`.

- Показать дружелюбный bottom sheet «Спасибо! В этот сбор можно снова помочь через Xч Yм».
- Не дублировать как ошибку.

### 4.2. Превентивная блокировка
Не давать тапнуть кнопку «Помочь», если из списка уже известен `next_available_at > now` (см. 2.2). 429 — последняя линия защиты.

## 5. Экран «Спасибо» + CTA подписки

### 5.1. Проверка активной подписки
- При открытии экрана/шторки «Спасибо» → дёрнуть `GET /subscriptions/active`.
- Если `has_active == false` → показать кнопку «Оформить подписку».
- Если `has_active == true` → не показывать (или показать «Управлять подпиской» вместо).

### 5.2. Поведение
- Тап по «Оформить подписку» ведёт на упрощённый экран подписки (см. 6).

---

## 6. Упрощённый экран оформления подписки

- Один экран, один CTA: «По рублю в день — раз в месяц спишем 30 ₽».
- Минимум полей: только выбор способа оплаты (см. 8) и подтверждение.
- Убрать лишние опции (выбор периодичности и т.д. — теперь только monthly по умолчанию).

---

## 7. Экран «Моя подписка» — модалка

- Открытие управления подпиской — теперь модалка/bottom sheet, а не отдельный экран.
- Содержит: сумма, дата следующего списания, кнопки «Приостановить», «Отменить», «Сменить карту».

## 8. Saved payment methods

### 8.1. Новый раздел в профиле
- «Способы оплаты» → список карт из `GET /payment-methods`.
- Каждая карта: `**** 1234`, тип (Visa/MC/МИР), бейдж «Основная».
- Действия: «Сделать основной», «Удалить».

### 8.2. Сохранение при первом донате
- На экране ввода карты — чекбокс «Сохранить карту для будущих платежей» (по умолчанию включён).
- Передавать `save_payment_method: true` в `POST /donations`.

### 8.3. Быстрая оплата
- Если у юзера есть сохранённые карты — на экране доната по умолчанию выбрана дефолтная.
- При тапе «Поддержать» сразу `POST /donations` с `payment_method_id`, без редиректа в YooKassa.
- Опция «Оплатить другой картой» открывает обычный flow.

## 9. Визуальные правки (из 2_meet.md)

- **Лого вместо тега** в шапке.
- **Поменять навбар** (детали в дизайне).
- **Кнопка «Подробнее»** располагается сразу после контейнера с текстом сбора.
- **Экраны документов**: галерея картинок документов (как в YouTube/Туба). Кнопка «Документы» из карточки сбора → экран со списком превьюшек документов. Источник: уже существующий `GET /campaigns/{id}/documents` (бэкенд готов).
- **Голосовое как в Telegram** — без стрелок (опциональная фича для записи спасибо-сообщения).
- **Конфетти + виброотклик** при успешном донате (длинный отклик). Реализуется флаттер-пакетами `confetti` и `flutter_haptic_feedback`/`vibration`.

## 10. Push-уведомления

- Регистрировать FCM-токен и отправлять его в `device-register` (или потом через `PATCH /profile`).
- Обрабатывать deep link из push «Можно снова поддержать» — открывать экран нужного сбора.
- В настройках профиля — переключатель «Напоминать о повторных донатах» (ключ `notification_preferences.push_on_donation_reminder`).
