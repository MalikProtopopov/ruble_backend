# Обзор проекта Fond: реализованный функционал и задел

Документ для исследования: что уже работает в **Flutter-приложении**, откуда взяты **мок-данные**, какие экраны **заглушки**, и что описано как **следующий этап** (бэкенд, платежи). Исходники: [flutter_app/](../../flutter_app/), референс UIKit: [legacy/Ruble/](../../legacy/Ruble/).

---

## 1. Контекст продукта

- **Назначение:** демо благотворительного клиента: вертикальная лента сборов с видео, детали сбора, «документы», флоу «помочь» с выбором суммы и способа оплаты (UI), шаринг, экран благодарности с видео и аудио, онбординг, нижняя навигация из трёх разделов.
- **Активный код:** только [flutter_app/](../../flutter_app/). Старый нативный проект перенесён в [legacy/](../../legacy/) как эталон дизайна и контента.
- **Стек:** Flutter, Riverpod (`ProviderScope`, `FutureProvider`, `StateProvider`), локальные ассеты (видео, шрифты Gilroy, thanks-медиа), `shared_preferences`, `video_player`, `just_audio`, `url_launcher`, `share_plus`.
- **Данные:** без сервера; интерфейсы репозиториев реализованы моками. Смена на API описана в [backend_roadmap.md](backend_roadmap.md).

## 3. Экраны и пользовательские сценарии (что реально делает приложение)

### 3.1 Главная — вертикальная лента сборов (`HomeScreen` + `CampaignFeedPage`)

- Данные: `campaignsProvider` → `MockCampaignRepository.fetchCampaigns()`.
- **Важно:** в моке в ленту отдаётся только **первые 5** кампаний из внутреннего списка из **8** записей (`take(5)`). Остальные три кампании доступны только через `fetchCampaign(id)` теоретически, в UI ленты не показываются.
- На каждой карточке: фоновое **видео из assets** (`videoAssetPath`), градиент снизу, заголовок, прогресс сбора (`raisedFunds` / `needFunds`), кнопки «Помочь», «Подробнее», иконка шаринга.
- Воспроизведение: играет только **активная** страница `PageView`; при открытии fullscreen thanks все видео на паузе (`pauseAllVideosProvider`).

### 3.2 Детали сбора (`CampaignDetailSheet`)

- Открывается снизу (`DraggableScrollableSheet`): заголовок, текст описания с раскрытием («Узнать больше» / «Скрыть»), блок «Подтверждающие документы» → кнопка открывает список документов.

### 3.3 Документы (`DocumentListSheet`)

- Список из `Campaign.documents` (мок: одни и те же три пункта на всех кампаниях).
- По тапу: `url_launcher` открывает **внешнюю ссылку**. Сейчас для всех документов используется **один и тот же тестовый PDF URL** (константа в `mock_campaign_repository.dart`), не уникальные файлы по сбору.

### 3.4 Донат (`DonateSheet` + `SelectPayMethodSheet` + `AddCardSheet`)

- Выбор суммы: чипы **1 / 2 / 5 / 10 ₽** с поясняющими текстами (как в iOS DonatesModel).
- Способ оплаты: **SberPay**, **СБП**, **Банковская карта**. Карта ведёт в форму `AddCardSheet` (номер, срок, CVC) с **минимальной валидацией**; при «успехе» возвращается в донат с подставленным суффиксом `*4242` (демо).
- Кнопка оплаты: **нет реального платежа** — `Future.delayed(3 секунды)`, затем закрытие шита и вызов `onCompleted(campaignId)` → открытие thanks.
- Визуально для SberPay кнопка с зелёным фоном; для остальных — градиент.

### 3.5 Шаринг (`ShareSheet`)

- Опции: «Сторис», «В ленту» — вызывают `share_plus` с текстом и **фиктивной ссылкой** `https://example.com/campaign`.
- «Скопировать ссылку» — копирует тот же stub в буфер, snackbar «Ссылка скопирована», закрытие шита.
- **Не привязан** к конкретной кампании: один и тот же URL для всех.

### 3.6 Благодарность (`ThanksFullscreen`)

- После «успешной» оплаты: `MockThanksRepository.contentForCampaign(campaignId)`.
- Если контента нет — экран не показывается, видео снова играют.
- Если есть: полноэкранно **видео + аудио** (ассеты из `assets/thanks/`), заголовок/описание из мока, управление воспроизведением аудио.

### 3.7 Вкладка «Волна» (`ConnectionsScreen`)

- Карточка «Авторизуйся…» + кнопка «Авторизоваться» — **`onPressed: () {}`** (ничего не делает).
- Блок «Столько раз вы стали причиной помощи» + число **140** — захардкожено, не из репозитория.
- Секция «Поделись сборами»: четыре одинаковых `ListTile` «Поделиться» — **`onTap: () {}`**.

## 4. Мок-данные (источник правды в коде)

### 4.1 Кампании — `MockCampaignRepository`

- Файл: [mock_campaign_repository.dart](../../flutter_app/lib/data/repositories/mock_campaign_repository.dart).
- Поля модели `Campaign`: `id`, `title`, `videoAssetPath`, `raisedFunds`, `needFunds`, `description`, `documents`.
- **8 кампаний** в статическом списке; тексты и сюжеты — медицинская благотворительность (как в legacy iOS interactor’ах).
- `fetchCampaigns()` возвращает **5** элементов — осознанное ограничение ленты.
- Документы: `DocumentItem(title, link)` — три названия на русском, **один общий URL** на PDF (внешний учебный пример, не продакшен).

### 4.2 Thanks — `MockThanksRepository`

- Файл: [mock_thanks_repository.dart](../../flutter_app/lib/data/repositories/mock_thanks_repository.dart).
- Список `ThanksContent` привязан к `campaignId` **1–5**.
- Для неизвестного `campaignId` метод **`contentForCampaign` возвращает третью запись** (fallback на индекс 2), то есть для кампаний 6–8 при «донате» в демо может показаться благодарность «не той» кампании, если бы они попали в ленту (сейчас в ленте только 1–5, так что в типичном сценарии совпадает).

### 4.3 Подключение моков к приложению

- [app_providers.dart](../../flutter_app/lib/core/providers/app_providers.dart): `campaignRepositoryProvider` и `thanksRepositoryProvider` жёстко указывают на мок-классы. Замена на HTTP — отдельная задача (см. [backend_roadmap.md](backend_roadmap.md)).

## 5. Что задумано, но не реализовано как продуктовая логика

Сводка по коду и документу следующей фазы:

| Область | Сейчас | Задел / намерение |
|---------|--------|-------------------|
| **Backend / API** | Нет сети | Реальные репозитории на `dio`/`http`, `baseUrl`, авторизация |
| **Платежи** | Имитация задержкой 3 с | SDK банка/агрегатора, СБП, SberPay, карты; не хранить PAN/CVC |
| **Медиа** | Только `assets/` | Видео/аудио по URL с CDN, при необходимости кэш |
| **Модели** | Простые Dart-классы | `fromJson` / совместимость с контрактом API |
| **Авторизация** | Кнопка без действия | Сохранение статистики донатов (копия из iOS Connections) |
| **Волна — шаринг списком** | Пустые `onTap` | Социальный/реферальный сценарий из референса |
| **Вкладка «Цели»** | Заглушка | Отдельный раздел продукта (не портирован) |
| **Шаринг** | Один stub URL | Диплинки / уникальная ссылка на сбор |
| **Ошибки сети** | Не применимо | Централизованная обработка (в legacy были нюансы с completion) |

Подробный чеклист следующей фазы: [backend_roadmap.md](backend_roadmap.md).

## 8. Краткий вывод

Приложение — **полноценный UI-прототип** благотворительной ленты с локальными видео, имитацией оплаты и экраном благодарности, на **зашитых мок-данных** (5 сборов в ленте, 8 в репозитории, общие ссылки на «документы», один stub для шаринга). Разделы **«Волна»** и **«Цели»** либо статичны, либо заглушки. Логика **сервера, платежей, авторизации и настоящих ссылок** вынесена в план следующей фазы и пока **не реализована**.
