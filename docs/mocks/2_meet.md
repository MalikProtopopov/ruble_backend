## Доработки проекта «По Рублю» — требования по итогам встречи с заказчиком

---

### 1. Анонимная авторизация (Auto-register)

**Проблема:** Обязательная почта при донате снижает конверсию — не все готовы сразу её указывать.

**Решение заказчика:** Убрать привязку почты как обязательное условие. Реализовать **авто-регистрацию устройства** при первом запуске приложения (или после истечения токена).

**Логика:**
- При инициализации приложения вызывается эндпоинт типа `POST /auth/device-register`
- Бэкенд создаёт профиль пользователя и возвращает токен
- Все последующие запросы (включая платежи) идут с этим токеном
- У такого пользователя статус: **почта не верифицирована**

**Привязка почты (опционально, позже):**
- Пользователь может в любой момент указать почту (регистрация или вход)
- Бэкенд привязывает почту к существующему профилю по токену
- Все предыдущие данные **наследуются** на новый аккаунт с верифицированной почтой:
  - Разовые платежи
  - Подписки
  - Сохранённые методы оплаты

**Срок жизни токена:** настраиваемый, предположительно ~6 месяцев.

---

### 2. Сохранение методов оплаты

- Необходимо сохранять карты пользователя
- Цель — в будущем поддержать **быструю оплату в 2 клика** (СБП / saved cards)
- Пользователь не должен вводить карту каждый раз заново

---

### 3. Ограничение частоты пожертвований

**Задача:** Ограничить пользователя по времени в рамках одного сбора, чтобы он не мог вносить взносы бесконечно.

**Логика:**
- Отслеживать последнее пожертвование пользователя **по токену** в конкретном сборе
- Сравнивать время последнего взноса с текущим
- Возвращать клиенту информацию: **можно ли сейчас сделать новый взнос**

**Предполагаемое ограничение:** раз в **6–8 часов** в одном и том же сборе.

---

### 4. Доработка списка сборов (эндпоинт)

В ответ эндпоинта активных сборов добавить дополнительные ключи для авторизованного пользователя:

| Ключ | Описание |
|---|---|
| `preview_image_url` | Ссылка на превью-изображение сбора |
| `donated_today` | Было ли пожертвование в этот календарный день |
| `last_donation` | Объект последнего пожертвования пользователя в этом сборе (или `null`) |
| `next_available_at` | Время, когда пользователь сможет снова сделать взнос в этом сборе |
