# API: документы сбора (для мобильного приложения)

Базовый URL: `https://backend.porublyu.parmenid.tech`
Префикс: `/api/v1/campaigns`

Авторизация **не требуется** — оба эндпоинта публичные. Документы возвращаются
только для сборов в статусе `active`.

---

## 1. Список документов сбора

Получить все документы конкретного сбора.

```
GET /api/v1/campaigns/{campaign_id}/documents
```

**Path-параметры**

| Параметр      | Тип    | Описание           |
|---------------|--------|--------------------|
| `campaign_id` | UUID   | ID сбора           |

**Ответ `200 OK`** — массив документов, отсортированный по `sort_order` (по возрастанию):

```json
[
  {
    "id": "0192f0c1-7a3e-7c11-9a2b-1f3e4d5c6b7a",
    "title": "Финансовый отчёт за 2025 год",
    "slug": "finansovyy-otchet-za-2025-god",
    "file_url": "https://cdn.porublyu.ru/docs/report-2025.pdf",
    "sort_order": 0
  },
  {
    "id": "0192f0c1-7a3e-7c11-9a2b-2a4f5e6d7c8b",
    "title": "Договор оферты",
    "slug": "dogovor-oferty",
    "file_url": "https://cdn.porublyu.ru/docs/offer.pdf",
    "sort_order": 1
  }
]
```

Если у сбора нет документов — вернётся пустой массив `[]`.

**Ошибки**

| Код   | Когда                                              |
|-------|----------------------------------------------------|
| `404` | Сбор не найден или не в статусе `active`            |

## 2. Детальная страница документа по slug

Получить один конкретный документ сбора по его `slug` (значение приходит в списке выше).

```
GET /api/v1/campaigns/{campaign_id}/documents/{slug}
```

**Path-параметры**

| Параметр      | Тип    | Описание                                   |
|---------------|--------|--------------------------------------------|
| `campaign_id` | UUID   | ID сбора                                   |
| `slug`        | string | slug документа (из поля `slug` в списке)   |

**Пример запроса**

```
GET /api/v1/campaigns/0192f0c1-.../documents/finansovyy-otchet-za-2025-god
```

**Ответ `200 OK`** — один объект документа:

```json
{
  "id": "0192f0c1-7a3e-7c11-9a2b-1f3e4d5c6b7a",
  "title": "Финансовый отчёт за 2025 год",
  "slug": "finansovyy-otchet-za-2025-god",
  "file_url": "https://cdn.porublyu.ru/docs/report-2025.pdf",
  "sort_order": 0
}
```

**Ошибки**

| Код   | Когда                                              |
|-------|----------------------------------------------------|
| `404` | Сбор не найден / не активен, либо документ с таким `slug` не существует у этого сбора |

---

## Модель документа

| Поле         | Тип     | Описание                                                        |
|--------------|---------|-----------------------------------------------------------------|
| `id`         | UUID    | Уникальный идентификатор документа                              |
| `title`      | string  | Название для отображения                                        |
| `slug`       | string  | URL-идентификатор, уникален **в рамках сбора**                  |
| `file_url`   | string  | Прямая ссылка на файл (PDF и т.п.) — открывать/качать по ней    |
| `sort_order` | integer | Порядок сортировки в списке (меньше = выше)                     |

## Заметки для интеграции

- **`slug` уникален только внутри сбора**, не глобально. Всегда обращайся к документу
  в паре с `campaign_id`: `/{campaign_id}/documents/{slug}`.
- `slug` генерируется автоматически из названия (транслитерация с кириллицы), поэтому
  он человекочитаемый и подходит для deeplink/шаринга. При совпадении названий внутри
  одного сбора добавляется суффикс `-2`, `-3` и т.д.
- Поле `slug` теперь приходит и в `documents[]` внутри детального ответа сбора
  `GET /api/v1/campaigns/{campaign_id}` — можно строить переход на детальную
  страницу документа без дополнительного запроса списка.
- Сам файл отдаётся по `file_url` напрямую (CDN), а не через этот API.
